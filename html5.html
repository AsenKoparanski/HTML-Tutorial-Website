HTML5 page structure

What's different in HTML5?

HTML5 has a changed and simplified DOCTYPE and offers new semantics which are block-level elements.

HTML5 DOCTYPE:
<!DOCTYPE html>

The <header> tag which defines the heading of the page.
<header>HEADER</header>

The <navigation> tag which defines a set of navigation links/menus.
<nav>NAVIGATION</nav>

The <section> tag defines a section part of the page and has its own heading.
<section>SECTION</section>

The <article> tag defines self-contained content and should make sense on its own. Can be nested with section both ways.
<article>ARTICLE</article>

The <aside> tag defines content that is placed aside from the main content, like a sidebar. It should also be related to the surrounding content information.
<aside>SIDEBAR</aside>

The <footer> tag defines the footer for a document and typically cotaints information about the author, copyright, links, etc.
<footer>FOOTER</footer>

Other interesting features

The <dialog> tag is used to mark up a conversation
<dialog>conversation</dialog>

The <figure> and <figcaption> tags add a visual explanation to an image.
<figure> <img src="image.jpg" > <figcaption>Caption of image.</figcaption> </figure>

The <canvas> tag draws a rectangular are on the HTML page and by default it has no border or content but it can be styled with CSS and javascript to draw graphics with it. Here is what a simple HTML example of this would look like:
<canvas id="Canvas" width="200" height="100"></canvas>
<canvas id="Canvas1" width="200" height="100"></canvas>
<canvas id="Canvas2" width="200" height="100"></canvas>



Style Guide

Always use lower case element names. This is correct and will validate but it is bad practice:
• Don't do:
<HEADER>Text</HEADER>
• Instead do:
<header>Text</header>

Always close all elements. In HTML5 you don't have to close them but it is recommended following best practice:
• Don't do:
<p>Text
• Instead do:
<p>Text</p>

Try using lower case names for attributes because it looks cleaner and is easier to write. Mixing lowercase and uppercase is bad.
• Don't do:
<p CLASS="text">Text</p>
• Don't do:
<p cLaSs="text">Text</p>
• Instead do:
<p class="text">Text</p>

HTML5 allows you to add attributes without quotes. It is recommended to still quote values because it is easier to read and if the value containts spaces it will not work.
• Won't work:
<table class=table striped>
• Don't do:
<table class=striped>
• Instead do:
<table class="striped">

HTML5 allows spaces around the equal signs but it is better practice to put as less as you can because it is easier to read:
• Don't do:
<p class = "text" >Text</p>
• Instead do:
<p class="text">Text</p>
